## 3. Debug 记录

**报错现象：**

运行任务 4 高峰小时系数计算的代码时，程序在 `count_5min = peak_data.resample('5T').size()` 这一行抛出异常并中断运行，完整报错信息为：
ValueError: Invalid frequency: T. Failed to parse with error message: KeyError('T'). Did you mean min?


报错提示 resample 的时间频率参数无效，导致后续 5 分钟、15 分钟粒度的聚合都无法执行，PHF 系数计算完全无法进行。

**解决过程：**

1. **定位错误行**：首先根据报错堆栈定位到问题出现在 `resample()` 函数的频率参数 `'5T'` 上。初步检查拼写没有错误，确认是 pandas 版本兼容问题。
2. **排查根本原因**：查询本地 Python 环境，安装的是 pandas 2.2.3 最新稳定版。查阅 pandas 官方更新文档后确认：pandas 从 2.2 版本开始正式移除了 `'T'` 作为分钟频率的历史别名，统一使用 `'min'` 作为分钟的标准频率字符串，旧版本的 `'T'` 写法不再被支持，因此触发参数无效报错。
3. **代码修复**：将任务 4 中两处 resample 的频率参数分别修改：`'5T'` 改为 `'5min'`，`'15T'` 改为 `'15min'`。
4. **结果验证**：修改后重新运行代码，时间重采样聚合正常执行，输出的高峰小时为 09:00~10:00、刷卡量 14961 次、最大 5 分钟刷卡量 1424 次、PHF5=0.8755、PHF15=0.9212，所有数值均和题目案例完全匹配，验证修复正确。

**额外兼容优化**：顺便修复了任务 3 中 seaborn.barplot 的 FutureWarning（不传递 hue 参数直接使用 palette 的弃用警告），给 barplot 添加了 `hue='route_label'` 和 `legend=False` 参数，消除了运行警告，提升了代码在高版本 seaborn 下的兼容性。

## 4. 人工代码审查（逐行中文注释）

```python
# 按小时分组统计全天各小时的上车刷卡总量，得到索引为小时、值为对应刷卡量的Series
hour_total = df_board.groupby('hour').size()
# idxmax()返回刷卡量最大值对应的小时数，自动识别出全天客流最高的高峰小时
peak_hour = hour_total.idxmax()
# 获取高峰小时对应的实际刷卡总人次，作为后续PHF计算的分子
peak_volume = hour_total.max()

# 将高峰小时数值格式化为两位数字的时间字符串，例如09:00~10:00，用于控制台结果输出
peak_start_str = f"{peak_hour:02d}:00"
peak_end_str = f"{peak_hour + 1:02d}:00"
print(f"高峰小时: {peak_start_str}~{peak_end_str}, 刷卡量: {peak_volume}次")

# 筛选出高峰小时内的所有上车刷卡记录，并将交易时间列设置为行索引
# 设时间类型索引是pandas resample时间重采样的必要前提，只有时间索引才能按固定时间窗口聚合
peak_data = df_board[df_board['hour'] == peak_hour].set_index('交易时间')

# 使用resample按5分钟为时间窗口对高峰小时数据重采样，size()统计每个窗口内的刷卡记录数
count_5min = peak_data.resample('5min').size()
# 从所有5分钟窗口中提取最大的刷卡量数值，代表高峰小时内客流最集中的5分钟流量
max_5min_val = count_5min.max()
# 获取最大5分钟窗口对应的起始时间戳
max_5min_time = count_5min.idxmax()
# 将时间戳格式化为"时:分"的字符串形式，得到窗口的起始时间
max_5min_start = max_5min_time.strftime('%H:%M')
# 起始时间加上5分钟的时间长度，得到该窗口的结束时间，组成完整时间区间用于输出
max_5min_end = (max_5min_time + pd.Timedelta(minutes=5)).strftime('%H:%M')

# 按照交通工程高峰小时系数的标准公式计算PHF5
# 公式：PHF5 = 高峰小时实际刷卡量 / (1小时内5分钟窗口总数 × 最大5分钟刷卡量)
# 分母含义：假设高峰小时内每个5分钟窗口都达到最大流量时的理论小时总量
# PHF取值范围为0~1，数值越接近1说明高峰小时内客流分布越均匀，数值越小说明客流越集中在短时段
PHF5 = peak_volume / (12 * max_5min_val)

print(f"最大5分钟刷卡量({max_5min_start}~{max_5min_end}): {max_5min_val}次")
print(f"PHF5  = {peak_volume} / (12 × {max_5min_val}) = {PHF5:.4f}")

# 按15分钟为时间窗口对高峰小时数据重采样，统计每个窗口内的刷卡记录数
count_15min = peak_data.resample('15min').size()
# 获取15分钟粒度下的最大刷卡量
max_15min_val = count_15min.max()
# 获取最大15分钟窗口对应的起始时间戳
max_15min_time = count_15min.idxmax()
# 格式化输出15分钟时间窗口的起止时间
max_15min_start = max_15min_time.strftime('%H:%M')
max_15min_end = (max_15min_time + pd.Timedelta(minutes=15)).strftime('%H:%M')

# 计算15分钟粒度的高峰小时系数PHF15
# 分母中数字4的含义：1小时包含4个连续的15分钟时间窗口
# 通常PHF15的数值会比PHF5更接近1，因为时间粒度越大，短时波动被平滑得越明显，客流分布越均匀
PHF15 = peak_volume / (4 * max_15min_val)

print(f"最大15分钟刷卡量({max_15min_start}~{max_15min_end}): {max_15min_val}次")
print(f"PHF15 = {peak_volume} / (4 × {max_15min_val}) = {PHF15:.4f}")
